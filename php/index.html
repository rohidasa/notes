    HOW TO SET UP PACKAGE.JSON & VITE.CONFIG.JS

    Use the terminal, and type this to create the package.json: 
    npm init -y
    Then use this command to install the vite plugin: 
    npm install laravel-vite-plugin --save-dev
    If you still do not have the vite.config.js file, just create it manually, and then paste this inside the file: 
    import { defineConfig } from 'vite';
import laravel from 'laravel-vite-plugin';

export default defineConfig({
    plugins: [
        laravel([
            'resources/css/app.css',
            'resources/js/app.js',
        ]),
    ],
});

    Installing Docker on Windows

    1. Check System Requirements Before starting, ensure your PC meets these minimum specs: * **OS:** Windows 10 (Build 19045+) or Windows 11. * **RAM:** 4GB minimum (8GB recommended). * **BIOS:** **Hardware Virtualization** must be enabled (usually found under "Advanced" or "CPU Configuration" in your BIOS settings). 2. Install WSL 2 (The Engine) Docker Desktop uses WSL 2 as its backend for better performance. 1. Open **PowerShell** as an Administrator (Right-click Start > Terminal Admin/PowerShell Admin). 2. Type the following command and press Enter: ```powershell wsl --install ``` 3. **Restart your computer** when prompted. 4. After restarting, a Linux terminal (usually Ubuntu) will pop up. Follow the prompts to create a **username** and **password**. 3. Download and Install Docker Desktop 1. Go to the [Official Docker Desktop Download Page](https://www.docker.com/products/docker-desktop). 2. Click **Download for Windows**. 3. Run the `.exe` installer. 4. **Crucial Step:** When the "Configuration" window appears, ensure **"Use WSL 2 instead of Hyper-V (recommended)"** is checked. 5. Follow the prompts to finish the installation and click **Close and restart** (if required). 4. Final Configuration Once your PC is back up: 1. Launch **Docker Desktop** from your Start menu. 2. Accept the **Docker Subscription Service Agreement**. 3. In the Docker Desktop dashboard, go to **Settings (gear icon) > General** and verify that "Use the WSL 2 based engine" is selected. 4. Go to **Resources > WSL Integration** and toggle "Enable" for your Linux distribution (e.g., Ubuntu) if you plan to use the Linux terminal. --- 5. Verify the Installation To make sure everything is humming along perfectly, open **PowerShell** or **Command Prompt** and run: 
    docker run hello-world
    If you see a message saying **"Hello from Docker!"**, you are officially ready to containerize. > **Note:** If you are using Docker for a company with more than 250 employees or $10M in annual revenue, you'll eventually need a paid subscription (Docker Pro, Team, or Business).
