Unity3D: Preparing, Building & Releasing Games - Important Tips

Adaptive Performance Controller

Control the Power & Thermal settings using AdaptivePerformanceController.cs. For gameplay set OnDemandRendering.renderFrameInterval to 1; For menus and other screens set its value to 2 or 3.

Canvas & Camera: Culling Mask

Do not let 2 cameras render the same layer. Flash animations may cause trouble (though you set higher sorting orders to loading screen)

Always set layers to GameSceneHUD like "HudUI" or "GameSceneHUD" and set the same layer as cullingMask for the camera.

Set layers to GameScene like "UI" or something else and set the same layer as cullingMask for the camera.

Setup Guidelines for Push Notifications with OneSignal

  1. Create a Firebase account. Do not install SDK
  2. Create OneSignal account
  3. Link Firebase to OneSignal with sender ID and server key
  4. Download and install OneSignal APK using package manager (if installed properly, all setup files will be removed from Assets folder except link.xml)
  5. Add PushNotificationManager.cs component to the intro scene
  6. Init OneSignal on PlayFabManager.onPlayfabSetupCompleteEvent

Exit on Suspend for iOS (OnApplicationPause)

iOS applications are usually suspended and do not quit. For iOS builds, enable the "Exit on Suspend" property in Player Settings to make the application quit and not suspend. Otherwise, you will see calls to OnApplicationPause instead.

Install Target SDK (Android)

For anyone that finds this and is having problems with a Unity install not having an Android SDK when trying to build:

  1. Run CMD (command prompt) as administrator
  2. Navigate to the Android SDK Manager installed by Unity:
    cd C:\Program Files\Unity\Hub\Editor\[UNITYVERSION]\Editor\Data\PlaybackEngines\AndroidPlayer\SDK\tools\bin
  3. Run SDK Manager:
    sdkmanager.bat
  4. Install specific API version:
    sdkmanager "platform-tools" "platforms;android-30"
  5. Agree to terms: Y

Your missing SDK will now be installed. Close and restart Unity if you had it open.

Android Upload Error Fix: "You uploaded an APK or Android App Bundle which has an activity, activity alias, service or broadcast receiver with intent filter, but without 'android:exported' property set. This file can't be installed on Android 12 or higher."

Solution: Unity android app build for android 12 with android:exported="true" property

Note: If you're using Unity 2020, upgrade to 2021 first, then fix with the above solution.

Fix App Suspension on 'Update Latest Unity SDK' or 'Google Admob SDK'

Unity Ads SDK: In Package Manager, update the 'Advertisement' package to the latest version. Then, in Project Settings → Services, link the Unity Project with your Unity ID.

Admob SDK: Download the Google Admob SDK package file and import it to Unity. Update your AdManager according to new updates.

Fixing App Crashing on Startup in Android 12 and Above

Solution 1 (Unity 2020): Forum Discussion

Edit the mainTemplate.gradle file:

  1. Enable "Custom Main Gradle Template" in ProjectSettings > PlayerSettings > PublishingSettings > Build
  2. Update dependencies in mainTemplate.gradle with Google Play Services and WorkManager dependencies
  3. Set android.useAndroidX to true in gradleTemplate.properties

Solution 2: Uninstall Native Share Plugin

Preparing RC (Release Candidate) Build for Android

  1. Remove all music, backgrounds, and sprites related to the UI framework
  2. Update package name in Firebase (was set as com.okey.game.test)
  3. Reduce all music/SFX quality rate to 1
  4. Disable logs
  5. Remove Lunar console prefab
  6. Obfuscator (for copyrighted/unsafe games)
  7. Scrambling (for copyrighted/unsafe games)
  8. Intent redirection (for copyrighted/unsafe games)
  9. Rename game folder and confirm with developer (for copyrighted/unsafe games)
  10. Test: App icon should be the same in Console and installed devices

Build Report Tool Setup

  1. Install the "Build Report Tool" package
  2. Build with recommended settings:
    • Uncheck "Development build" in Player Settings
    • Set Compression method to "LZ4HC"
    • Set Minimum API Level to 23
    • Set Target API Level to 34
    • Use IL2CPP Scripting Backend
  4. After build, go to Window → Show Build Report
  5. Optimize unused textures with high quality compression and crunch compression
Important: Remove the "SimpleAndroidNotifications" folder as it's not supported in newest Android 12 devices.

Preparing AAB (Android App Bundle) Build

  1. Check "Build App Bundle (Google Play)" in Build settings
  2. Enable ARM64 under Target Architecture in Player Settings
  3. Disable logs in Player Settings (Stack Trace → Log Type to None)
  4. Set Debug.unityLogger.logEnabled = false; in scripts

Preparing iOS Build

Developer Account: Apple Developer Account

Video Tutorial

Step 1: Create a Bundle Identifier

  1. Go to "Identifiers" section
  2. Click add button (+ icon)
  3. Choose "App IDs" and continue
  4. Select type as "App" and continue
  5. Add description
  6. Add explicit Bundle Identifier (e.g., rummy.turkey.okey)
  7. Enable "App Groups" & "Push Notification" under Capabilities
  8. Click Register

Step 2: Create Provisioning Profile

A provisioning profile connects your App ID with signing certificates and authorized devices.

  1. Go to "Profiles"
  2. Click plus symbol
  3. Select "iOS App Development"
  4. Choose App ID and continue
  5. Select certificates and continue
  6. Select test devices and continue
  7. Name your provisioning profile and generate
  8. Download and double-click to add to Keychain

Step 3: Create Certificate (.p12)

  1. Go to "Certificates"
  2. Click plus icon to create new certificate
  3. Choose "Apple Push Notification service SSL (Sandbox & Production)"
  4. Create new CertificateSigningRequest from Mac Keychain
  5. Convert downloaded .cer file to .p12 certificate
  6. Upload certificate in OneSignal

Step 4: Create App on App Store Connect

  1. Log in to App Store Connect
  2. Go to "My Apps" and create new app
  3. Select iOS platform
  4. Enter app name and language
  5. Choose previously created Bundle ID
  6. Enter alphanumeric SKU
  7. Note down the iOS store app ID (e.g., id6445915638)
Important: Open \Assets\Editor\MediationAdapterDependencies.xml and comment out unity meditation pods inside <iosPods> tags.

iOS Player Settings and Build Settings

  1. Update App Tracking Transparency in Package Manager
  2. Update iOS store app ID with 'id' prefix (e.g., id6445915638)
  3. Update Bundle Identifier (e.g., rummy.turkey.okey)
  4. Update version and build number
  5. Remove "USE_IAP" scripting define if no in-app purchases
  6. Add microphone usage description: "Talk to your friends!"
  7. Build with Release configuration on Mac PC
  8. Run pod setup and pod install

If you encounter pod issues:

pod deintegrate
pod install --repo-update
