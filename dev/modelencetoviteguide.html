Modelence → Standalone Vite

Step-by-Step Migration Guide

Convert any downloaded Modelence project to run locally on http://localhost:3000

Overview

Modelence is a cloud-hosted full-stack platform. When you download a project it generates a scaffold that relies on the Modelence CLI (modelence dev), cloud authentication, MongoDB cloud, and proprietary client helpers. None of these are available when running locally without an internet connection to Modelence's servers.

This guide strips all Modelence dependencies and replaces them with standard Vite + React equivalents so the project runs at http://localhost:3000 with no external services required.

What Gets Replaced

createRoot().render() from react-dom

Prerequisites

Before starting, make sure you have the following installed on your machine:

  • Node.js v18 or higher (v20+ recommended). Check with: node --version
  • npm v9 or higher. Check with: npm --version
  • A code editor such as VS Code
  • The downloaded Modelence project folder (unzipped)

Step 1 — Understand the Folder Structure

A downloaded Modelence project looks like this. Note that the server files are often placed in the wrong directory (src/client/server/ instead of src/server/):

your-project/

  .modelence.env          ← cloud credentials (not needed locally)

  .npmrc                  ← Modelence registry config (must be removed)

  modelence.config.ts     ← tells Modelence where server entry is

  package.json            ← needs cleanup

  vite.config.js          ← corrupted on download — must be rewritten

  tsconfig.json           ← keep as-is

  src/

    client/               ← React frontend (keep everything here)

      server/             ← WRONG location — move this to src/server/

    server/               ← CORRECT location (create if missing)

⚠️ Important

The vite.config.js downloaded from Modelence is always corrupted — it is missing equals signs and has mangled import paths. Do not try to fix it; replace it entirely with a fresh vite.config.ts as shown in Step 3.

Step 2 — Move the Server Folder

Modelence expects server files at src/server/ but the download places them at src/client/server/. Fix this first.

On Windows (Command Prompt)

xcopy /E /I src\client\server src\server

rmdir /S /Q src\client\server

On Mac / Linux

mv src/client/server src/server

After this, your structure should be:

src/

  client/   ← frontend only

  server/   ← backend (app.ts, example/, migrations/)

Step 3 — Replace vite.config.js

Delete the existing vite.config.js entirely and create a new file named vite.config.ts with the following content:

import { defineConfig } from 'vite';

import react from '@vitejs/plugin-react';

import tailwindcss from '@tailwindcss/vite';

import path from 'path';

export default defineConfig({

  root: 'src/client',

  plugins: [react(), tailwindcss()],

  resolve: {

    alias: {

      '@': path.resolve(__dirname, './src'),

    },

  },

  server: {

    port: 3000,

    host: true,

  },

  build: {

    outDir: '../../dist',

    emptyOutDir: true,

  },

});

Step 4 — Rename seo.site.ts

The project imports from seo.config.ts but the downloaded file is named seo.site.ts. Rename it:

On Windows

rename src\client\seo.site.ts seo.config.ts

On Mac / Linux

mv src/client/seo.site.ts src/client/seo.config.ts

Step 5 — Delete .npmrc

The .npmrc file points npm to Modelence's private registry. Running npm install with it present will fail. Delete it:

On Windows

del .npmrc

On Mac / Linux

rm .npmrc

ℹ️ Note

The .modelence.env file can be left or deleted — it is ignored once the Modelence CLI is no longer used.

Step 7 — Create a Local Auth Context

Modelence provided useSession(), loginWithPassword(), signupWithPassword(), and logout() from modelence/client. Replace all of these with a single local file.

Create the file src/client/lib/auth.tsx:

import React, { createContext, useContext, useState, useEffect } from 'react';

export interface User {

  _id: string;

  email: string;

  handle: string;

}

interface AuthContextValue {

  user: User | null;

  isLoading: boolean;

  login: (email: string, password: string) => Promise<void>;

  signup: (email: string, password: string) => Promise<void>;

  logout: () => Promise<void>;

}

const AuthContext = createContext<AuthContextValue | null>(null);

const STORAGE_KEY = 'app:user';

export function AuthProvider({ children }: { children: React.ReactNode }) {

  const [user, setUser] = useState<User | null>(null);

  const [isLoading, setIsLoading] = useState(true);

  useEffect(() => {

    const stored = localStorage.getItem(STORAGE_KEY);

    if (stored) { try { setUser(JSON.parse(stored)); } catch {} }

    setIsLoading(false);

  }, []);

  const login = async (email: string, _password: string) => {

    const u: User = { _id: '1', email, handle: email.split('@')[0] };

    localStorage.setItem(STORAGE_KEY, JSON.stringify(u));

    setUser(u);

  };

  const signup = async (email: string, _password: string) => {

    const u: User = { _id: '1', email, handle: email.split('@')[0] };

    localStorage.setItem(STORAGE_KEY, JSON.stringify(u));

    setUser(u);

  };

  const logout = async () => {

    localStorage.removeItem(STORAGE_KEY);

    setUser(null);

  };

  return (

    <AuthContext.Provider value={{ user, isLoading, login, signup, logout }}>

      {children}

    </AuthContext.Provider>

  );

}

export function useSession() {

  const ctx = useContext(AuthContext);

  if (!ctx) throw new Error('useSession must be used inside AuthProvider');

  return ctx;

}

ℹ️ Note

This is a localStorage stub — it stores a fake user with no real password verification. It is suitable for local development of a marketing/frontend site. If you need real authentication later, replace the login/signup functions with calls to your own API.

Step 8 — Rewrite src/client/index.tsx

The original index.tsx calls renderApp() from modelence/client. Replace it with standard React 19 createRoot:

import { StrictMode, Suspense } from 'react';

import { createRoot } from 'react-dom/client';

import { RouterProvider } from 'react-router-dom';

import { QueryClient, QueryClientProvider } from '@tanstack/react-query';

import { Toaster } from 'react-hot-toast';

import { router } from './router';

import { AuthProvider } from './lib/auth';

import { Seo } from './components/Seo';

import LoadingSpinner from './components/LoadingSpinner';

import './index.css';

const queryClient = new QueryClient();

function App() {

  return (

    <Suspense fallback={<LoadingSpinner fullScreen />}>

      <Seo />

      <Toaster position="top-right" />

      <RouterProvider router={router} />

    </Suspense>

  );

}

const root = document.getElementById('root');

if (!root) throw new Error('No #root element found');

createRoot(root).render(

  <StrictMode>

    <QueryClientProvider client={queryClient}>

      <AuthProvider>

        <App />

      </AuthProvider>

    </QueryClientProvider>

  </StrictMode>

);

Step 9 — Fix index.html

Vite requires the entry script tag to point at index.tsx, not a compiled JS file. Replace the contents of src/client/index.html:

<!doctype html>

<html lang="en">

  <head>

    <meta charset="UTF-8" />

    <meta name="viewport" content="width=device-width, initial-scale=1.0" />

    <title>Your Site Name</title>

  </head>

  <body>

    <div id="root"></div>

    <script type="module" src="/index.tsx"></script>

  </body>

</html>

Step 10 — Fix router.tsx

The router imports useSession from modelence/client. Change that one import to your local auth context. Everything else in the router stays the same.

Find this line:

import { useSession } from 'modelence/client';

Replace it with:

import { useSession } from './lib/auth';

Step 11 — Fix Pages That Import from modelence/client

Search every file in src/client/ for imports from modelence/client and apply the fixes below.

Page.tsx

Change:

import { useSession } from 'modelence/client';

To:

import { useSession } from '@/client/lib/auth';

LoginPage.tsx

Remove:

import { getConfig, loginWithPassword } from 'modelence/client';

Add:

import { useSession } from '@/client/lib/auth';

Then replace the handleSubmit function to use login() from useSession() instead of loginWithPassword().

SignupPage.tsx

Remove:

import { signupWithPassword } from 'modelence/client';

Add:

import { useSession } from '@/client/lib/auth';

Then replace the handleSubmit function to use signup() from useSession().

LogoutPage.tsx

Remove:

import { logout } from 'modelence/client';

Add:

import { useSession } from '@/client/lib/auth';

import { useNavigate } from 'react-router-dom';

Replace the useEffect to call logout() from useSession() and then navigate to /login.

PrivateExamplePage.tsx

Change:

import { useSession } from 'modelence/client';

To:

import { useSession } from '@/client/lib/auth';

autoLogin.ts

This file uses Modelence sandbox auto-login. Replace the entire file with a no-op:

// Sandbox auto-login was a Modelence-only feature.

export function useAutoLogin() {}

Step 12 — Fix tsconfig.json

Remove the "next" plugin entry from tsconfig.json (it is a Modelence/Next.js specific plugin that causes errors with plain Vite). Also remove noUnusedLocals and noUnusedParameters if they cause build errors during migration.

Find and remove this block from compilerOptions.plugins:

  "plugins": [{ "name": "next" }]

Also remove if present:

  "noUnusedLocals": true,

  "noUnusedParameters": true,

  "noFallthroughCasesInSwitch": true

Step 13 — Install Dependencies and Run

Now delete the old node_modules (which were installed via Modelence's registry) and do a clean install:

On Windows

rmdir /S /Q node_modules

del package-lock.json

npm install

npm run dev

On Mac / Linux

rm -rf node_modules package-lock.json

npm install

npm run dev

Then open your browser at:

http://localhost:3000

Troubleshooting

Error

Fix

Cannot find module 'modelence/client'

You missed an import in one of the files. Search all .tsx/.ts files for "from 'modelence" and fix each one.

Cannot find module '@modelence/react-query'

Remove any imports of modelenceMutation or modelenceQuery. These require a Modelence backend.

ERR_MODULE_NOT_FOUND: src/server/app.ts

The server folder is still at src/client/server/. Redo Step 2.

Cannot find module 'seo.config'

The file is still named seo.site.ts. Redo Step 4.

npm install fails with 404 on @modelence/*

The .npmrc still points to Modelence registry. Redo Step 5.

Port 3000 already in use

Change the port in vite.config.ts (server.port) and package.json (--port flag).

Blank white page, no errors

Check that index.html has <div id="root"> and the script src points to /index.tsx.

TypeScript errors on unused vars

Remove noUnusedLocals and noUnusedParameters from tsconfig.json (Step 12).
